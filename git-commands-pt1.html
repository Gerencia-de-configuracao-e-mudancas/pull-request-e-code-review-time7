Comandos Git — Parte 1

Missão: Seus amigos devem clonar este repositório e apagar todos os nomes em vermelho (são os "bugs"). Depois deverão commitar e abrir um pull request. Boa sorte!

1) Inicializar um repositório

Comando: 

git init
Cria um repositório Git local no diretório atual.

Exemplo prático: iniciar um projeto criado por João Silva que esqueceu de remover seu nome.

2) Clonar um repositório remoto

Comando: 

git clone https://url-do-repositorio.git
Faz uma cópia local de um repositório remoto.

Se alguém cometeu acidentalmente um arquivo com autoria errada (por exemplo Mariana), você deverá removê-lo localmente e commitar a correção.

3) Adicionar mudanças ao stage

Comando: 

git add arquivo.txt
# ou para todos os arquivos
git add .
Prepara mudanças para o próximo commit.

Exemplo: remover a tag com o nome Carlos que é um bug no README.

4) Fazer um commit

Comando: 

git commit -m "Remover nomes de teste em vermelho"
Salva um snapshot das mudanças indexadas.

5) Verificar o estado do repositório

Comando: 

git status
Mostra arquivos modificados, staged, untracked etc. Use este comando antes de commitar para revisar o que será enviado.

6) Subir mudanças para o remoto

Comando: 

git push origin main
Envia seus commits para o branch remoto (ex.: main). Se alguém deixou um nome em vermelho como Felipe, este push incluirá a remoção após você commitar.

Observe os nomes em vermelho — eles são os "bugs". Apague-os e faça commit.
