Missão (continuação): Apague todos os nomes em vermelho também neste arquivo. Eles são os bugs a serem removidos.

7) Criar e trocar branches

Comando: 

git branch nova-feature
git checkout nova-feature
# ou em uma linha
 git switch -c nova-feature

8) Atualizar do remoto

Comando: 

git pull origin main
Puxa mudanças do remoto e tenta mesclar no branch atual.

9) Rebase e merge

Comando merge: 

git checkout main
git merge nova-feature
Rebase (reaplicar seus commits): 
git checkout nova-feature
git rebase main

10) Stash (guardar mudanças temporariamente)

Comando: 

git stash
# recuperar
git stash pop
Útil quando precisa trocar de branch sem commitar mudanças incompletas.

11) Reset e revert

Reset (mover HEAD): 

git reset --hard HEAD~1
Revert (cria um novo commit que desfaz outro): 
git revert

12) Tags e releases

Comando: 

git tag -a v1.0 -m "Versão 1.0"
git push origin v1.0
Marcar versões importantes do projeto.

Depois de apagar todos os nomes em vermelho: verifique com git status, faça git add, git commit e git push. Voltar para a Parte 1.
